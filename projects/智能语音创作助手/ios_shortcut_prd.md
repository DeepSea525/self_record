# iOS 快捷指令：AI 多模态多平台文案创作 (VibeCoding PRD)

> **背景与目的**: 本文档专为大模型“VibeCoding（对话式编程/提示词编程）”设计。你可以将这份文档直接“喂”给 Cursor、Claude 等工具，让其一步步指导你拼装 iOS 快捷指令（Shortcuts），或直接输出相关的 AppleScript / JavaScript for Automation 脚本（如果需要的话，但通常快捷指令更依赖具体的拖拽步骤指导）。

## 一、 产品概述

- **产品名称**: AI 多模态文案助手 (iOS 快捷指令)
- **核心功能**: 用户在手机端快速输入图文内容，快捷指令自动调用**火山引擎（大模型）API**，根据用户选择的目标平台（如小红书、大众点评），针对性地生成符合平台调性的专属文案，并自动保存到备忘录和剪贴板。

## 二、 核心流程与功能规格

### 1. 初始化配置与鉴权 (API Key 存储)

- **需求描述**: 用户首次运行该快捷指令时，需要进行一次性配置，后续无需重复操作。
- **技术实现建议 (Shortcuts)**:
  - 检查 iCloud 云盘特定路径（如 `Shortcuts/AI_Copywriter/config.txt`）是否存在 API Key。
  - **如果存在**: 直接读取该文本作为 `API_KEY` 变量。
  - **如果不存在**: 弹出一个“请求输入”对话框：“请输入您的火山引擎 API Key”。获取输入后，使用“存储文件”动作将其保存在上述路径，并赋值给 `API_KEY` 变量。

### 2. 目标平台选择 (Platform Selection)

- **需求描述**: 运行后，第一时间通过菜单让用户选择本次要创作的目标平台。
- **MVP 版本支持平台**:
  - 【大众点评】
  - 【小红书】
- **技术实现建议**: 使用快捷指令的“从菜单中选取”动作。根据用户的选择，设置一个名为 `Platform` 的变量。

### 3. 用户主输入区 (图文收集)

- **需求描述**: 收集用户本次需要大模型处理的原始素材（文本 + 图片，最多5张）。
- **技术实现建议**:
  - **文本输入**: 使用“请求输入”操作，提示语为：“简单描述一下你要发的素材（例如：今天吃了顿火锅，味道很一般，但是服务很好）”。存入 `UserText` 变量。
  - **图片选择**: 使用“选择照片”操作。
    - 允许选择多张照片（开启该选项）。
    - 限制最多选择 5 张图片。
  - **图片预处理 (关键)**: 遍历选中的每一张图片，调整图片大小（如限制在 1024 像素以内以节省 API tokens 和避免请求超时），然后使用“Base64 编码”操作将图片转为文本。由于火山引擎大模型 API 需要 base64 格式的图片，将这些 Base64 字符串存入一个字典或数组供后续调用。

### 4. 平台专属 Prompt 提取 (核心大脑)

- **需求描述**: 结合用户的个人创作价值观，为不同的平台注入不同的系统提示词 (System Prompt)。
- **Prompt 设计指导 (融入了博主的[十条内容创作指导原则])**:
  
  **【小红书 System Prompt】**
  > “你是一个在小红书拥有极高粘性粉丝的干货/生活博主。你现在需要帮我把原始素材扩写/改写为一篇小红书爆款图文笔记。
  > 你的创作必须严格遵循以下原则：
  > 1. 展现活人感：融入第一人称的情感、笨拙的探索过程，要真实，像闺蜜/兄弟在聊天。
  > 2. HKR原则结构：标题和首段必须有 Hook（抓人眼球）；正文必须有 Knowledge（提供高于90%读者的信息增量或避坑指南）；结尾必须有 Resonance（情感共鸣）。
  > 3. 用故事包装：将评测或感悟转化为‘冲突 -> 探索 -> 结局’的结构。
  > 4. 格式规范：使用 Emoji 点缀，多用短句，核心重点加粗，文末添加相关且精准的 Hashtag。
  > 请根据我提供的图片内容和粗糙的文字，生成这一篇笔记。”

  **【大众点评 System Prompt】**
  > “你是一个资深的大众点评Lv8达人。你需要将我的原始素材改写为一篇优质的探店/拔草评价。
  > 你的创作必须严格遵循以下原则：
  > 1. 绝对真诚与中立：优点缺点必须明确表态，绝不夸大其词，做最客观真实的评价。
  > 2. 结构清晰：分为这几个维度来写：【环境与服务】、【核心菜品/体验细节】、【总结与避坑/推荐指数】。
  > 3. 信息密度高：把读者当成聪明且忙碌的人，不讲废话，直击核心利益点（如怎么停车、怎么点单最划算）。
  > 4. 结合图片描述：请仔细观察提供的图片，并在文字中呼应图片中的特色（例如菜品色泽、环境氛围）。
  > 请结合我的文字诉求和图片，生成一条不少于 150 字的高质量评价。”

### 5. 火山引擎 API 调用 (API Request)

- **API 选择**: 需要使用火山引擎支持**多模态分析**的大模型（如 `Doubao-pro-vision` 等支持视觉的模型）。
- **技术实现建议**:
  - 使用快捷指令的“获取 URL 内容”动作。
  - 方法：`POST`。
  - 请求头 (Headers): `Content-Type: application/json` 和 `Authorization: Bearer <API_KEY>`。
  - 请求体 (Body): 构建符合火山引擎 API 规范的 JSON 数据。将对应的平台 System Prompt 作为 `role: system`，将用户的文字 `UserText` 以及 Base64 编码的图片数组作为 `role: user` 的消息体传入。

### 6. 输出结果与本地流转

- **需求描述**: 获取 API 返回后，进行文本解析，并将生成的文案存储到用户的剪贴板和备忘录中。
- **技术实现建议**:
  - 使用“获取词典值”解析 API 返回的 JSON，提取 `choices[0].message.content`（具体依火山引擎 API 响应结构为准）。
  - 使用“拷贝至剪贴板”动作：将提取到的文案直接复制，方便用户立刻打开对应的 App 粘贴。
  - 使用“创建备忘录”动作：将生成的文案以及当天的日期创建为一条新的 Apple Notes（苹果备忘录），方便日后查阅和归档。
  - 使用“显示通知”或“显示提醒”动作：向用户弹出“✅ 文案创作完成！已复制到剪贴板并存入备忘录。”的完成反馈。
