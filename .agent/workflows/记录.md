# 记录内容并分类

当用户向你描述日常记录时（例如"今天发生了xxx"或"我想记录xxx"），按以下步骤处理：

## 步骤

1. **解析内容**
   - 识别记录的时间（如用户未提供，使用当前时间）
   - 提取关键信息、情绪、涉及的领域

2. **识别输入类型** ⭐ 关键步骤
   判断这条记录属于哪种类型：

   **A. 项目驱动型输入**
   - 特征：为了完成某个明确目标而进行的学习、调研、行动
   - 关键词：为了、用于、准备、xxx项目、下一步要...
   - 归档：`projects/[项目名]/` 目录

   **B. 纯粹沉淀型输入**
   - 特征：信息积累，没有明确的项目目的，但可能将来有用
   - 关键词：学到了、看到了、想到了、觉得有意思...
   - 归档：`areas/` 或 `knowledge/` 目录

   **如何判断？** 问用户："这条记录是为了某个具体项目，还是单纯的信息积累？"

3. **确定分类**
   根据输入类型和内容判断归属：

   **项目驱动型** → `projects/[项目名]/log.md` 追加记录

   **纯粹沉淀型** →
   - `areas/工作/` - 工作相关的信息
   - `areas/学习/` - 学习、技能提升相关
   - `areas/生活/` - 日常生活、人际关系
   - `areas/健康/` - 运动、饮食、睡眠、心理健康
   - `knowledge/` - 方法论、认知框架、个人原则
   - `inbox/` - 无法明确分类的内容

3. **生成文件名**
   格式：`YYYY-MM-DD-简短描述.md`
   例如：`2024-12-16-项目复盘思考.md`

4. **创建文件**
   使用以下结构：

   ```markdown
   # [标题]
   
   > 记录时间: YYYY-MM-DD HH:MM
   > 分类: [领域]
   > 标签: #标签1 #标签2
   
   ## 内容
   [用户描述的内容，可适当整理格式]
   
   ## 所思所想
   [用户的思考和感悟]
   
   ---
   *由AI整理于 {{TIMESTAMP}}*
   ```

5. **告知用户**
   告知用户文件已创建及其位置

## 注意事项

- 如果涉及多个领域，选择最主要的一个
- 如果用户提到的是项目相关内容，询问是否需要创建新项目或归入现有项目
- 保持原始表达，不要过度修改用户的语言风格
